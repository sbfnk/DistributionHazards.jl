ContactIntervals.jl

A small shared kernel: the parametric distribution families for the contact interval as introduced by Kenah (2011), the time from the onset of infectiousness in one individual to infectious contact with another. The hazard function h(τ) of a contact interval is the infectiousness kernel β(τ) at age-of-infection τ.

This package owns just the typed distribution families and their hazard / cumulative hazard / survival / density (with log-scale variants), plus one derived helper, expected_cumhazard. Likelihoods, fitting, and simulation drivers live in downstream consumers like TransStat.jl.

See NOTICE.md for attribution and references; distributed under LGPL ≥ 3.

At a glance

using ContactIntervals

d = WeibullCI(0.4, 1.6)               # rate λ = 0.4, shape γ = 1.6

hazard(d, 2.0)                        # h(τ)
cumhazard(d, 2.0)                     # H(τ) = ∫₀^τ h
survival(d, 2.0)                      # S(τ) = exp(-H(τ))
density(d, 2.0)                       # f(τ) = h(τ) S(τ)

log_hazard(d, 2.0)                    # AD-stable
log_survival(d, 2.0)                  # = -cumhazard(d, t)
log_density(d, 2.0)

# Standard Distributions.jl interface comes for free.
using Distributions
pdf(d, 2.0); cdf(d, 2.0); quantile(d, 0.95); rand(d)

# R₀ scaffold: expected cumulative hazard over an infectious-period distribution.
expected_cumhazard(d, Exponential(1.0))    # E[H(ι)] via QuadGK

The four families

TypeCumulative hazard H(τ)
ExponentialCI(λ)λ·τ
WeibullCI(λ, γ)(λ·τ)^γ
LogLogisticCI(λ, γ)log(1 + (λ·τ)^γ)
GammaCI(λ, γ)-log Q(γ, λτ) (regularised upper incomplete gamma)

The rate/shape parameterisation follows the transtat/transreg lineage. Adding more families is by subtyping ContactInterval; each concrete type defines cumhazard, hazard, log_hazard, and Distributions.quantile. The rest is derived once at the abstract layer.

See the API reference for the full method list.

Validation

Pkg.test() runs ~930 unit tests. Highlights:

  • Definitional consistency (cumhazard ≡ -log_survival; density ≡ hazard·survival).

  • Full Distributions.jl interface (pdf/cdf/ccdf/logpdf/logccdf/quantile).

  • ForwardDiff through cumhazard and log_hazard w.r.t. rate and shape, including the t = 0 limit (for left-truncated data).

  • Cross-language: 660 kernel-level comparisons against R's transtat::transreg.distributions over a parameter grid. Worst absolute discrepancies:

    Familylog_hazardcumhazard
    exponential0.0 (bit-equivalent)0.0 (bit-equivalent)
    weibull0.0 (bit-equivalent)~1 ulp
    log-logistic~1 ulp~1 ulp

    i.e. machine precision; the differences come from using log(exp(x)) where R uses x directly, and log1p(x) where R uses log(1 + x).
